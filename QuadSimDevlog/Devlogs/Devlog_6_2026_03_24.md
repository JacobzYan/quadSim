So.... it looks like my breaks between these logs has gotten longer. It's been an eventful month so to summarize quickly, since the last log I have:

- Resolved some circular dependencies in my includes (as you can see with this wonderful map of what inherits what)![[Pasted image 20260324225619.png]]
As you may be able to see (and if you do see it, you're smarter than me), there is an include loop around sensorClass, quadParams, and quadState. These all need to include eachother because:
- Sensors need to take in the drone's state in order to produce a reading
- the quadParams class needs to store a list of pointers to sensors so the estimator knows what sensors to query
- the quadState has to hold params in order to track what state variables exist (such as IMU biases, and number of props)
The first thing I thought of was to just shove all of these datatypes into 1 file, but it turns out the smarter way to do it is to forward declare the sensor class into quadParams since it just needs to hold a reference to a class. I had thought before that in order to hold an object as a member variable in a class, you had to include a reference to the actual object in the header of the class. Turns out you just need something with the same name (especially if you're not going to be interacting with that member object directly), yay learning!

That was a fun learning adventure that took up a good portion of the month (C++ sure has some strangely phrased error messages), but I am now to the point that I have a tester for the forward ODE simulation written, and I just have some simple compile time errors to debug. Hopefully the inevitable logical errors in the implementation aren't too bad. I guess we'll have to see later!
