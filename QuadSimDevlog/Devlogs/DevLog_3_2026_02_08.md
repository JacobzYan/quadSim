Woah its been a while, most of the last week (2 weeks?) has just been spent getting the params function to work. Its a little embarrasing that it took me this long, but in my defense, I have been busy with work, and I've been picking up a lot of small C++ details that I've never had to worry about being a predominantly python/MATLAB user. 

The greatest hits are:
- Actually understanding what .cc and what .h files are, and what goes in them
- Learning about the difference between pointers and references, what each does and when to pass them in
- Future proofing my CMake setup so that now all my testers go into their own self contained folders in /sandbox
- Learning about streaming strings for setting up file parsing

I still have a long way to go for sure, but today was a milestone for the fact that I was able to get a moderately spread out executable (tester for the quadParams class) to successfully compile and run! It doesn't pull the data in correctly right now but thats just a matter of debugging the implementation which is something I can do, Horray!

Also coming up is the question of whether I want to integrate my dependencies as git submodules, or use Cmake's FetchContent. I am currently leaning a bit more towards submodules just because I've had to wipe my build directory a couple of times to get CMake to stop caching stuff, but that could also be solved by just using cmake . --fresh as well. Maybe there's some funny automation that I can do with copying the library to a temp folder, wiping build, then moving it back. Things to think about.....

One last topic is pushing this to github: I have not done so yet, because I plan to include godot as part of the visualizer, and I want to make sure I have a good handle on what needs to go in the .gitignore so I can avoid a messy commit tree (and so I can figure out how setup should go for other people). Plus, pushing this to github will mean that this very devlog will go public meaning that I won't just be talking to myself like a crazy person anymore! That's a plus right? I'm not expecting anyone to actually read this (except maybe one of my friends that I mention this to), but the knowledge that this is out there and that someone **could** read it will be enough motivation to stay strong on this project!
