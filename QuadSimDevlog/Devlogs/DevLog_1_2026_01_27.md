Hello anyone who reads this, This is part of my plan to increase my accountability in terms of working on this project. 

Here's some background to start (hopefully this portion of the project will also help me with my writing and communication skills in general):

I did my MSE in dynamic systems and controls with a heavy lean towards robotics. I am currently working doing mechanical design, but I would love the opportunity to work in robotics or GNC in the future. Unfortunately, I don't really have a project I could talk about in any potiential interviews, so I fear that if I were to apply for a job in one of those fields, I would have a difficult time talking about my skills. Plus everyone knows that you don't really know a field till you've tried to make something work in it which brings us here! To be able to keep my my skills up, and build experiments/a project for any potential future interviews, I am attempting to write my own flight controller from the ground up for an autonomous quadcopter. 

Why do I think I can do this? Good question, in my last semester of my MSE, I took ASE 479W - Aerial Robotics at UT which focused on this exact idea: building up a flight controller from scratch! Now this may sound like cheating (it can't be very hard to write a flight controller and simulator if you've already written a flight controller and simulator), but the codebase I generated in this class has a couple of issues:
1. We wrote our flight controller in MATLAB which means I don't have a great way to run the code now that my student license has expired :( MATLAB code is also relatively slow, and high level which means I don't have a great way to transfer it to a real quadcopter. I have also been interested in embedded devices recently, and this project is a good framework to explore that area more.
2. The class moved at a very fast pace (as it needed to in order to cover all the necessary ground), so there were several concepts that I wanted to explore that we were not able to. Off the top of my head, I can think of:
	1. Model correction and /or self calibration
	2. Navigation sans expensive GPS setups
	3. SLAM
3. There are also several experiments I'm interested in that require some hardware tinkering that we were not able to do in class such as:
	1. Adding the capability to carry items or mount a low DOF gripper of some kind
	2. Implementing an automatic charging setup either through an inductive charging setup or base station

So that's my background, motivation, and a list of some of my goals, we'll see if the project crashes and burns, or if I'm able to make something cool out of it!

For now, I'm working on making a simulator, and visualization tools. The reason for this is that a digital twin of a quad is much cheaper to crash than a real quadcopter. having a low risk accurate simulator will allow me to prototype new features/controller architectures. Once that's done, I plan to create a visualizer in Godot, and have the executable run the quad in real time. 

So far, I've laid out the project's file structure, and done a proof of concept simulation of a simple controller with an inverted pendulum sim (found in the sandbox folder of src). I've also worked on some automation tools such as a shell script that can do a run and visualize the data with python (automations/runTestOdeAndVisualize.sh)


That's all for this time (its midnight and I have work tomorrow), but I hope to keep this log going to keep me motivated, accountable, and somewhat organized.
